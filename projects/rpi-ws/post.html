RPIZW Web Server

Headless Pi USB Interface

Summary

A local web server is a very convenient for developing, debugging, and troubleshooting different web prototypes. At the time, I had a crappy outdated Macbook Pro that was no match for running MySQL/Apache 24/7 and I was working in IT for a local municipality that allowed me to repurpose/reuse some of the old equipment that were being disposed of bi-yearly. I chose an RJ11 tester as my enclosure and threw in some stickers I had laying around to create this ugly piece of hardware! This monstrosity is essentially a Raspberry Pi Zero with USB SSH Interfacing enabled. I am able to hit my endpoints via hostname.local and access my files through my editor (spacemacs) via DIRED/FTP for seemless file transfer/editing.

Installation

There are a cluster of 3 Pi's. Each have an LED that would be connected to a 5 ohm resistor connected in parallel to open GPIO headers. The LEDs would indicate start/stopped services on a bi-modal setting activated by the momentary push button. The original RJ11 test board had its parts stripped and was drilled to accommodate the Pi's standoffs. It has a square panel cut out in the back to access the micro-sd card easily and an internal USB storage for web development files in case the micro-sd card fails.
